= 更换控制器的要求 - AFF A400
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
在更换 AFF A400 系统的控制器之前，请验证系统中的所有其他组件是否正常工作，确保您有正确的更换控制器，并将控制器的控制台输出保存到文本日志文件中。

查看更换控制器模块的要求。

[NOTE]
====
NetApp 强烈建议您在使用的设备上启用控制台日志记录，并在执行此过程时执行以下操作：

* 在维护期间保持 AutoSupport 启用状态。这允许 NetApp 支持在发生问题时接收诊断信息。
* 为任何 CLI 会话启用会话日志记录。这将创建维护活动期间所有命令和系统输出的完整记录。有关如何启用会话日志记录的指导，请参见 https://kb.netapp.com/on-prem/ontap/Ontap_OS/OS-KBs/How_to_configure_PuTTY_for_optimal_connectivity_to_ONTAP_systems["如何配置 PuTTY 以优化与 ONTAP 系统的连接"^]。


====
* 所有驱动器架都必须正常工作。
* 健康的控制器必须能够接管被替换的控制器（在本程序中称为受损控制器）。
* 请勿使用此方法进行控制器升级。请参阅 https://docs.netapp.com/us-en/ontap-systems-upgrade/choose_controller_upgrade_procedure.html["选择您的控制器硬件升级操作步骤"]供参考。
* 如果您的系统采用MetroCluster配置，请查看 https://docs.netapp.com/us-en/ontap-metrocluster/disaster-recovery/concept_choosing_the_correct_recovery_procedure_parent_concept.html["选择正确的恢复操作步骤"]确定是否采用此程序。
* 用从NetApp收到的现场可更换单元 (FRU) 替换故障组件。
* 更换控制器模块时，请使用相同型号的控制器模块。您无法通过更换控制器模块来升级系统。
* 在此过程中，您不能更换驱动器或驱动器架。
* 在此操作步骤 中，启动设备将从受损控制器移至更换控制器，以便更换控制器在与旧控制器模块相同版本的 ONTAP 中启动。
* 理解本流程中使用的控制器术语：
+
** 受损的控制器是指将被替换的控制器。
** 替换控制器是指替换故障控制器的新控制器。
** _health_ 控制器是运行正常的控制器。


* 使用  `security key-manager onboard show-key-store` 检查是否配置了板载密钥管理器 (OKM)。如果是这样，请在开始此过程之前使用  https://docs.netapp.com/us-en/ontap-cli/security-key-manager-onboard-show-backup.html["security key-manager onboard show-backup"] 检索集群范围的密码短语或联系 NetApp 支持人员。
